How it works

Three steps from spec to screen.

You write the story. The engine drives a real browser, records it, and composes the finished film — voiceover, captions, and camera included.

01 · WRITE

A declarative spec

Scenes in YAML: a line of narration and one action — go, click, type, scroll. Diffable and reviewable like any other file.

02 · CAPTURE

Drive & record

Playwright performs the flow with an animated cursor while recording — and notes exactly where every click lands.

03 · COMPOSE

Frame & finish

Studio framing, zoom-to-click, voiceover, captions, music — composed into an mp4 with an .srt transcript beside it.

What's in the frame

Every detail of a hand-made demo, automated.

Studio framing

The recording floats on a gradient backdrop inside a macOS browser window — rounded, shadowed, real.

Zoom that follows

Brand & themes

The whole demo, in one file

Readable enough to review in a pull request.

onboarding.yaml
title: "Cold start"
url: "http://localhost:3000"
preset: studio
scenes:
  - narrate: "Open the console."
    goto: "/guide"
  - narrate: "Start a new thread."
    click: "text=New Thread"
  - narrate: "Ask a question."
    type: { selector: "#composer",
            text: "Who are my top clients?" }
  - narrate: "Seconds later — the answer."
    press: "Enter"
    spotlight: "#results"   # the payoff

Out comes a 1080p film.

Framed on a gradient, narrated, captioned, and zoomed to the moment that matters — with a transcript beside it.

  • Verify selectors against the live page before rendering
  • Preview fast, then render the final cut
  • One spec → mp4 · srt · vtt · transcript
Install

Drop it in as a Claude Code skill.

Mount the repo, install once, and ask Claude Code to make a demo — or drive the engine yourself from the CLI.

# add the skill to your project
$ git submodule add https://github.com/our-nature/demoreel .claude/skills/demo
$ cd .claude/skills/demo/engine
$ uv sync --extra piper && uv run playwright install chromium

# …then, in Claude Code
› /demo a walkthrough of the onboarding flow

# …or drive the engine directly
$ uv run demoreel render onboarding.yaml -o out.mp4
